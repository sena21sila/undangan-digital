Format Link Undangan

Tentang NanoID
NanoID diimplementasikan langsung dalam JavaScript tanpa library eksternal apapun. Menggunakan crypto.getRandomValues() - generator acak kriptografis bawaan browser. Dengan 64 karakter alfabet dan panjang 10 karakter, kemungkinan tebak acak adalah 1 banding 1,15 x 10^18 - mustahil secara praktis.

Bagian 03

Setup Google Apps Script

Apps Script menjadi backend API gratis yang menghubungkan kedua halaman HTML ke Google Sheets. Ikuti langkah berikut dengan seksama - hanya perlu dilakukan sekali.

1

Buat Google Spreadsheet baru

Buka sheets.google.com → klik + Blank → beri nama bebas, contoh: "Database Undangan".

Info
Sheet tambahan (Tamu, Pengaturan, RSVP, Ucapan) akan dibuat otomatis oleh Apps Script saat pertama kali dijalankan.
2

Buka Editor Apps Script

Di Spreadsheet → klik menu Extensions → pilih Apps Script. Tab baru editor kode akan terbuka.

3

Tempel kode dari apps-script.js

Di editor, tekan Ctrl+A lalu Delete untuk hapus semua kode yang ada. Buka file apps-script.js dengan Notepad, salin semua isinya, tempel ke editor.

WAJIB - Ganti Secret Key
Cari baris ini dan ganti dengan kata sandi buatanmu:

const SECRET_KEY = "GANTI_DENGAN_SECRET_KEY_KAMU";

Gunakan minimal 12 karakter kombinasi huruf besar/kecil, angka, dan simbol.
Contoh: Nik4h@nBudi2026!

Simpan dengan Ctrl+S.

4

Deploy sebagai Web App

Klik tombol Deploy (kanan atas) → New deployment. Isi konfigurasi:

TypeWeb app
Execute asMe (akun Google kamu)
Who has accessAnyone ← WAJIB

Klik Deploy → izinkan akses saat diminta → salin URL yang muncul.

URL yang perlu disalin
https://script.google.com/macros/s/AKfycb.../exec
Jika muncul error
Pastikan "Who has access" sudah dipilih Anyone, bukan "Only myself".
5

Verifikasi berhasil

Buka URL deployment di browser. Jika berhasil akan muncul:

{"ok":false,"msg":"Action tidak dikenal: undefined"}

Itu normal - artinya Apps Script aktif dan siap menerima request.

Bagian 05

Uji Coba di Localhost

Jangan buka file HTML langsung dari folder (protokol file://) karena browser akan memblokir fetch API, audio, dan clipboard. Gunakan local server sederhana.

A

VS Code Live Server (Paling Mudah)

  • Buka folder proyek di VS Code
  • Install extension Live Server oleh Ritwick Dey
  • Klik kanan undangan.html → Open with Live Server
  • Browser otomatis buka: http://127.0.0.1:5500/undangan.html
B

Python HTTP Server

Buka terminal di folder proyek, jalankan:

python -m http.server 8080

Buka browser: http://localhost:8080/undangan.html

C

Node.js

npx http-server -p 8080

Buka browser: http://localhost:8080/undangan.html

Test Proteksi Link di Localhost
Karena halaman sekarang membutuhkan NanoID valid di URL, buka dengan format:

http://localhost:8080/undangan.html?to=abcdefghij

Pastikan tepat 10 karakter dari A-Z a-z 0-9 _ - . Tanpa parameter ini halaman akan menampilkan "Undangan Tidak Ditemukan".

Bagian 06

Cara Menggunakan Dashboard

1

Login

Buka dashboard.html → masukkan username admin dan password undangan2026. Salah 5 kali berturut-turut → dikunci 60 detik.

2

Hubungkan ke Google Sheets

Isi URL Apps Script dan secret key di kotak setup bagian atas → klik Hubungkan. Status berubah menjadi "Terhubung" jika berhasil. Koneksi tersimpan di browser, hanya perlu dilakukan sekali.

3

Isi Pengaturan Acara

Tab Pengaturan → isi semua detail: nama pasangan, tanggal, waktu, tempat, base URL undangan, URL Maps, dan data dua rekening → klik Simpan ke Sheets. Data ini tampil otomatis di halaman undangan.

Base URL Undangan
Isi dengan URL halaman undangan tanpa parameter, contoh:
https://username.github.io/undangan-nikah/undangan.html
Sistem otomatis menambahkan ?to=NanoID untuk setiap tamu.
4

Tambah Tamu (Manual atau Import)

Manual: isi nama dan nomor WA → klik Tambah. Setiap tamu mendapat NanoID 10 karakter unik otomatis.

Import massal: klik "Import CSV / Excel" → upload file → cek preview → klik "Import Sekarang". Format nomor WA: 6281234567890 (tanpa + dan spasi).

Untuk membuat file import, klik "Unduh Template CSV" terlebih dahulu → isi data → upload kembali.

5

Kirim Undangan via WhatsApp

Satu tamu: klik ikon preview (mata) untuk cek pesan → klik tombol WA untuk buka WhatsApp Web dengan pesan terisi otomatis.

Massal: centang beberapa tamu → klik "Kirim WA Dipilih" → setiap tamu dibuka di tab baru dengan jeda 0,7 detik.

6

Filter dan Monitor Tamu

Gunakan kolom pencarian untuk cari nama tamu. Gunakan dropdown Status Kirim untuk filter yang sudah/belum dikirim WA. Gunakan dropdown Filter RSVP untuk filter berdasarkan konfirmasi kehadiran tamu.

7

Export Data ke Excel

Klik tombol "Export Excel" di atas tabel tamu. File daftar-tamu-undangan.xlsx akan terdownload berisi semua data tamu lengkap dengan status RSVP masing-masing.

8

Ganti Kredensial Login

Tab Keamanan → isi username baru, password baru (min. 8 karakter), konfirmasi password → klik Simpan. Berlaku segera.

Bagian 07

Detail Fitur Baru

Animasi Fold-Unfold

Cover terdiri dari 4 panel (atas, bawah, kiri, kanan) yang membuka ke luar secara bersamaan saat tombol "Buka Undangan" diklik, menciptakan efek amplop yang dibuka. Menggunakan CSS 3D transform dengan perspective dan rotateX/rotateY. Tidak butuh library JavaScript apapun.

Dark / Light Mode

Diimplementasikan dengan CSS custom properties (var()) dan atribut data-theme di tag html. Toggle antara tema gelap (default) dan terang. Preferensi disimpan di localStorage browser.

NanoID - Generator ID Pendek

Implementasi mandiri tanpa library eksternal. Menggunakan crypto.getRandomValues() (Web Crypto API bawaan browser) untuk menghasilkan angka acak yang kriptografis. Alfabet 64 karakter (A-Z a-z 0-9 _ -) dengan panjang 10 karakter menghasilkan sekitar 1,15 x 10^18 kemungkinan kombinasi unik.

// Implementasi yang digunakan (pure JavaScript, no library)
function nanoid(){
  const chars='ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789_-';
  const arr=new Uint8Array(10);
  crypto.getRandomValues(arr);
  return Array.from(arr).map(b=>chars[b%64]).join('');
}

Import CSV / Excel

Menggunakan library SheetJS yang dimuat dari CDN hanya saat dibutuhkan (lazy loading). Mendukung file .csv, .xlsx, dan .xls. Deteksi kolom nama dan nomor WA otomatis berdasarkan nama header (tidak case-sensitive). Tampilkan preview 5 baris sebelum import untuk verifikasi.

Tombol Google Calendar

Membuka URL Google Calendar dengan parameter yang sudah terisi otomatis dari data pengaturan. Format tanggal di-parse dari format Indonesia ("Rabu, 25 Maret 2026"). Tidak memerlukan Google Calendar API atau OAuth - murni URL parameter standar yang didukung Google Calendar.

Troubleshooting

Muncul "Undangan Tidak Ditemukan" padahal link sudah benar

Validasi sekarang membutuhkan tepat 10 karakter dari alfabet A-Z a-z 0-9 _ - . Pastikan link disalin dari kolom Link di dashboard, bukan diketik manual. Jika tamu lama memiliki UUID panjang, tambahkan ulang sebagai tamu baru untuk mendapat NanoID.

Animasi fold-unfold tidak muncul / langsung skip

Pastikan halaman dibuka via local server (bukan file://). Cek juga apakah ada error di browser console (F12). Animasi menggunakan CSS 3D transform yang membutuhkan hardware acceleration aktif di browser.

Import CSV gagal - kolom tidak terdeteksi

Pastikan baris pertama file adalah header dan mengandung kata "Nama" di salah satu kolomnya. Kata "WA", "HP", "No", atau "Telepon" untuk kolom nomor. Header tidak case-sensitive. Download template CSV dari dashboard sebagai referensi format yang benar.

Tombol kalender tidak berfungsi / membuka halaman salah

Pastikan data tanggal di pengaturan sudah tersimpan ke Sheets dengan format yang benar: "Hari, DD Bulan YYYY", contoh: "Rabu, 25 Maret 2026". Nama bulan harus dalam Bahasa Indonesia. Coba simpan ulang pengaturan dari dashboard.

Export Excel kosong atau tidak ada data RSVP

Data RSVP di export diambil dari cache yang dimuat saat membuka tab RSVP di dashboard. Buka tab RSVP terlebih dahulu sebelum klik Export Excel agar data RSVP ikut ter-export.

Setelah update kode Apps Script tidak ada perubahan

Setiap perubahan kode harus di-deploy ulang dengan deployment baru: klik Deploy → New deployment (bukan edit yang lama). URL berubah dan perlu diupdate di dashboard serta di field SCRIPT_URL pada undangan.html.
